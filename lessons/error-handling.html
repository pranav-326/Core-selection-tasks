CORS explained

CORS (Cross-Origin Resource Sharing) is a browser security feature that restricts requests from one origin to another unless the server permits it via response headers (e.g., Access-Control-Allow-Origin).

How to fix CORS during development

  • Configure server to include CORS headers (for json-server use middleware)
  • Run mock API and site on same host/port (or serve via proxy)
  • Use a server-side proxy when deploying (Netlify functions, Vercel serverless, or custom proxy)
