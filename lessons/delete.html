Delete: DELETE and Optimistic UI

Learning objectives

  • Confirm destructive actions
  • Implement optimistic delete and rollback strategies
  • Handle server failures gracefully

Optimistic delete pattern

// OPTIMISTIC DELETE: remove UI before server; rollback on failure
function optimisticDeleteUI(id) {
  const el = document.querySelector(`[data-id="${id}"]`);
  const backup = el.cloneNode(true);
  el.remove();
  fetch(`${API_BASE}/posts/${id}`, { method: 'DELETE' })
    .then(res => {
      if (!res.ok) {
        // rollback
        document.querySelector('#posts').appendChild(backup);
      }
    })
    .catch(() => {
      // rollback on network error
      document.querySelector('#posts').appendChild(backup);
    });
}

Always show an "Undo" option or error banner so users can recover.
