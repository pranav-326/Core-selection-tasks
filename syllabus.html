Syllabus

Learning objectives

  • Explain REST, HTTP verbs and status codes
  • Implement robust fetch flows with retries
  • Perform CRUD operations against a mock API
  • Manage Authorization headers and tokens
  • Explain and remediate CORS issues

Schedule (approx)

  1. 0–10m: Intro, goals, environment
  2. 10–25m: REST basics and Postman demo - rest-basics
  3. 25–40m: json-server setup and data modeling - mock-api-and-setup
  4. 40–60m: Read/Create flows (GET/POST) - read, create
  5. 60–85m: Update & Delete (PUT/PATCH/DELETE) - update, delete
  6. 85–100m: Auth, external APIs, CORS - auth-and-external
  7. 100–120m: Error handling, retries, optimistic updates - error-handling
  8. 120m+: Demo, Q&A, deployment (GitHub Pages / Vercel / Netlify)

Materials

Use the interactive demo and the `db.json` mock API for hands-on practice.
